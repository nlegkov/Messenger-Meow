# CdM-16 Messenger — Документация проекта

Мессенджер для нескольких пользователей на учебном процессоре **CdM-16**,
реализованный в симуляторе Logisim с UART-периферией.

---

## Структура проекта

```
cdm16_messenger/
├── server.asm       — исходный код сервера на ассемблере CdM-16
├── router.py        — Python-роутер (мост TCP ↔ Logisim UART)
├── messenger.circ   — схема Logisim (инструкция по сборке внутри)
├── microcode.m      — микрокод для компилятора synthm
└── README.md        — этот файл
```

---

## Архитектура системы

```
Пользователь A                 Python Router              Logisim
  telnet :7001  ──────────►  :7001 ←→ :8001  ◄────────► UART #0 ─┐
  telnet :7002  ──────────►  :7002 ←→ :8002  ◄────────► UART #1 ─┤─► CdM-16
  telnet :7003  ──────────►  :7003 ←→ :8003  ◄────────► UART #2 ─┤  (сервер)
  telnet :700N  ──────────►  :700N ←→ :800N  ◄────────► UART #N ─┘
```

**Python Router** — работает на хосте, слушает входящие TCP-соединения от
пользователей (telnet) и проксирует трафик в UART-слоты Logisim. Каждый
пользователь привязан к одному UART-слоту.

**CdM-16** в Logisim — сервер чата. Принимает байты от UART через прерывания
(IRQ), накапливает строки в буферах, рассылает сообщения всем подключённым
клиентам.

## Карта памяти CdM-16 (Гарвардская модель)

### Память программы (ROM)

| Адрес     | Содержимое                        |
|-----------|-----------------------------------|
| `0x0000`  | Таблица векторов (9 записей × 4Б) |
| `0x0024`  | Зарезервировано                   |
| `0x0200`  | Код сервера (`main`)              |
| `0x1000+` | Строковые константы               |

### Память данных (RAM + MMIO)

| Адрес          | Содержимое                                        |
|----------------|---------------------------------------------------|
| `0x0000–0x00FF`| Стек (256 байт, растёт от 0x0100 вниз)            |
| `0x1000–0x11FF`| RX-буферы: 4 слота × 64 байта                    |
| `0x2000–0x203F`| Буферы имён: 4 слота × 16 байт                   |
| `0x3000–0x307F`| Broadcast-буфер (128 байт)                        |
| `0xE000–0xE007`| UART #0 (порт 8001)                               |
| `0xE010–0xE017`| UART #1 (порт 8002)                               |
| `0xE020–0xE027`| UART #2 (порт 8003)                               |
| `0xE030–0xE037`| UART #3 (порт 8004)                               |

## Регистры UART (MMIO)

Каждый UART занимает 16 байт начиная с базового адреса `0xE000 + slot×0x10`.

| Смещение | Тип | Название   | Описание                                         |
|----------|-----|------------|--------------------------------------------------|
| `+0x00`  | W   | TX         | Записать байт для отправки пользователю          |
| `+0x02`  | R   | RX         | Прочитать принятый от пользователя байт          |
| `+0x04`  | R   | STATUS     | Биты: [0]=con, [1]=dt, [2]=tx_ready              |
| `+0x06`  | W   | CTRL       | Биты: [0]=en_irq, [1]=clear_rx                   |

**Биты STATUS:**
- `con` (бит 0) — пользователь подключён к UART-порту
- `dt` (бит 1) — в буфере RX есть байт для чтения
- `tx_ready` (бит 2) — TX-буфер готов принять следующий байт

## Таблица векторов прерываний

| Номер | Адрес    | Назначение                  |
|-------|----------|-----------------------------|
| 0     | `0x0000` | Точка входа (старт/сброс)   |
| 1     | `0x0004` | Исключение: невыровн. sp    |
| 2     | `0x0008` | Исключение: невыровн. pc    |
| 3     | `0x000C` | Исключение: неверн. инструк.|
| 4     | `0x0010` | Double fault                |
| 5     | `0x0014` | IRQ от UART #0              |
| 6     | `0x0018` | IRQ от UART #1              |
| 7     | `0x001C` | IRQ от UART #2              |
| 8     | `0x0020` | IRQ от UART #3              |

## Протокол общения пользователя с сервером

```
Пользователь подключается → сервер ожидает строку-имя
Пользователь вводит имя + Enter → сервер сохраняет имя
Всем: "<имя> joined"
Сервер каждому: "Welcome, <имя>!\r\n"

Далее: пользователь вводит сообщение + Enter
Всем кроме отправителя: "<имя>: <сообщение>\r\n"
```

## Алгоритм работы сервера (asm)

```
main:
  init_stack()
  memzero(rx_bufs)
  memzero(name_bufs)
  for slot in 0..3:
    uart_init(slot)   ; включить IRQ
  ei()                ; разрешить прерывания
  loop:
    wait              ; ждать IRQ (режим ожидания)

IRQ от UART #N:
  читаем STATUS → проверяем con и dt
  если нет con → помечаем слот как disconnected
  если dt → читаем байт из RX
  если байт != '\n' → добавляем в rx_buf[N]
  если байт == '\n':
    если имя не задано → регистрируем (name_buf[N] = rx_buf[N])
      broadcast_join(N)
      send_welcome(N)
    иначе → broadcast_message(N)
    очищаем rx_buf[N]

broadcast_message(sender):
  формируем "<name>: <msg>\r\n" в bcast_buf
  для каждого слота != sender:
    если connected → uart_send_bytes(slot, bcast_buf)
```

## Структура RX-буфера слота (64 байта)

```
rx_buf[slot]:
  [0..61]  — накопленные байты сообщения (62 байта)
  [62]     — счётчик накопленных байт (1 байт)
  [63]     — флаг connected (0/1)
```

## Сборка схемы в Logisim

### Необходимые компоненты

1. **Logisim** — скачать с официального сайта
2. **logisim-cdm-emulator.jar** — из архива `cdm-devkit-misc/jar/`
   (репозиторий `cdm-processors/cdm-devkit`)
3. **logisim-uart.jar** — из репозитория `cdm-processors/logisim-uart`

### Шаги

1. Открыть `messenger.circ` в Logisim
2. Загрузить плагины: Project > Load Library > JAR Library
3. Собрать схему согласно инструкции внутри `messenger.circ`
4. Скомпилировать прошивку:
   ```bash
   cocas --target cdm16 server.asm -o server.img
   ```
5. Загрузить прошивку в ROM схемы
6. Запустить симуляцию в Logisim

### Схема соединений (упрощённо)

```
         ┌─────────────────────────────────────────┐
         │              Logisim                     │
         │                                          │
  ROM ───┤inst_data                                 │
  RAM ───┤data_in ──────────────────── CdM-16       │
         │data_out ─────── RAM/MMIO                 │
         │addr ─────────── ROM/RAM/MMIO             │
         │memory ──────────────────────────         │
         │data/inst ──────────────────────          │
         │read/write ─────────────────────          │
         │word ───────────────────────────          │
         │                                          │
         │irq ◄─── OR(dt0|dt1|dt2|dt3)             │
         │int_vec ◄── PriorityEncoder(dt0..dt3)     │
         │                                          │
  UART0 ─┤ port=8001 ──────────────── MMIO 0xE000  │
  UART1 ─┤ port=8002 ──────────────── MMIO 0xE010  │
  UART2 ─┤ port=8003 ──────────────── MMIO 0xE020  │
  UART3 ─┤ port=8004 ──────────────── MMIO 0xE030  │
         └─────────────────────────────────────────┘
```

## Запуск системы

### Шаг 1 — Запуск Logisim

```bash
java -jar logisim.jar messenger.circ
```

Нажать Run (Ctrl+K), убедиться что схема симулируется.

### Шаг 2 — Запуск роутера

```bash
python3 router.py
```

Вывод:
```
[12:00:00] INFO Slot 0: listening for users on port 7001
[12:00:00] INFO Slot 1: listening for users on port 7002
[12:00:00] INFO Slot 2: listening for users on port 7003
[12:00:00] INFO Slot 3: listening for users on port 7004
[12:00:00] INFO Router started. Connect with: telnet 127.0.0.1 <port>
```

### Шаг 3 — Подключение пользователей

На разных компьютерах (или терминалах):

```bash
# Пользователь 1
telnet 192.168.1.100 7001

# Пользователь 2
telnet 192.168.1.100 7002

# Пользователь 3
telnet 192.168.1.100 7003
```

### Шаг 4 — Использование

```
Connecting to 192.168.1.100...
Enter your name: Alice        ← вводим имя
Welcome, Alice!

Bob: Hello everyone!          ← сообщения от других
Alice: Hi Bob!                ← наше сообщение
```

## Ограничения текущей реализации

| Параметр              | Значение | Изменить в                        |
|-----------------------|----------|-----------------------------------|
| Макс. клиентов        | 4        | `MAX_CLIENTS` в server.asm        |
| Длина сообщения       | 62 байта | `BUF_SIZE` в server.asm           |
| Длина имени           | 15 байт  | `NAME_SIZE` в server.asm          |
| Только ASCII          | да       | расширить кодировку               |
| Нет истории           | —        | добавить кольцевой буфер          |

Увеличение `MAX_CLIENTS` требует добавления UART-компонентов в Logisim
и новых IRQ-векторов в таблицу.
